# FASTAPI INTENT CLASSIFICATION API

A PROFESSIONAL INTENT CLASSIFICATION API BUILT WITH **FASTAPI** AND **SCIKIT-LEARN**. THIS PROJECT PROVIDES A ROBUST ENDPOINT FOR PREDICTING USER INTENT FROM TEXT MESSAGES.

---

## KEY CONCEPTS & IMPLEMENTATION DETAILS

TO HELP YOU UNDERSTAND HOW THIS API WORKS, HERE ARE SOME KEY EXPLANATIONS:

### WHAT IS AN API?
AN **API (APPLICATION PROGRAMMING INTERFACE)** IS A SET OF RULES THAT ALLOW DIFFERENT SOFTWARE PROGRAMS TO TALK TO EACH OTHER. IT ACTS AS A MIDDLEMAN BETWEEN THE USER (CLIENT) AND THE MACHINE LEARNING MODEL (SERVER). THE CLIENT SENDS A **REQUEST** (E.G., A TEXT MESSAGE), AND THE API RETURNS A **RESPONSE** (THE PREDICTED INTENT).

### WHAT IS FASTAPI?
**FASTAPI** IS A MODERN, HIGH-PERFORMANCE WEB FRAMEWORK FOR BUILDING APIS WITH PYTHON. IT IS KNOWN FOR ITS:
- **SPEED**: AS FAST AS NODEJS AND GO.
- **VALIDATION**: AUTOMATIC DATA CHECKING USING PYDANTIC, WHICH PREVENTS INCORRECT DATA FROM BREAKING THE APP.
- **AUTO-DOCS**: IT AUTOMATICALLY GENERATES INTERACTIVE DOCUMENTATION (`/DOCS` AND `/REDOC`).

### HOW IS THE MODEL LOADED?
IN `APP.PY`, THE MACHINE LEARNING MODEL IS **PERSISTENT**, MEANING IT IS TRAINED ONCE AND SAVED AS A FILE. WE LOAD IT INTO MEMORY WHEN THE SERVER STARTS USING **JOBLIB**:
```python
# LOADING AT STARTUP ENSURES WE DON'T RELOAD FOR EVERY REQUEST
model = joblib.load("intent_model.pkl")
vectorizer = joblib.load("vectorizer.pkl")
```
BY LOADING THE MODEL AT THE TOP LEVEL OF THE CODE, IT REMAINS IN THE SERVER'S RAM, MAKING PREDICTIONS EXTREMELY FAST FOR EVERY SUBSEQUENT REQUEST.

### FLOW OF PREDICTION INSIDE THE API
WHEN A USER SENDS A MESSAGE TO THE `/PREDICT` ENDPOINT, THE FOLLOWING STEPS OCCUR:
1. **REQUEST**: THE USER SENDS A JSON PAYLOAD: `{"MESSAGE": "HELLO"}`.
2. **VALIDATION**: PYDANTIC CHECKS IF THE INPUT IS VALID.
3. **PREPROCESSING**: THE MESSAGE IS CLEANED AND STRIPPED OF EXTRA SPACES.
4. **VECTORIZATION**: THE `VECTORIZER` TRANSFORMS THE TEXT INTO A NUMERICAL VECTOR.
5. **INFERENCE**: THE `MODEL` PREDICTS THE INTENT BASED ON THAT VECTOR.
6. **RESPONSE**: THE API RETURNS THE RESULT: `{"INTENT": "GREETING"}`.

## GETTING STARTED

### 1. INSTALLATION
INSTALL THE REQUIRED DEPENDENCIES:
```bash
pip install -r requirements.txt
```

### 2. RUNNING THE API
START THE SERVER USING `UVICORN`:
```bash
uvicorn app:app --reload
```
THE API WILL BE AVAILABLE AT `HTTP://127.0.0.1:8000`.
